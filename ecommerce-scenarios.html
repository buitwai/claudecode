Inventory Management Scenarios

4 Scenarios

Scenario 1: Real-Time Inventory Sync

Beginner 3-4 hours

Background

AutoZone operates 6,000+ stores with millions of SKUs. The e-commerce platform needs real-time inventory synchronization to prevent overselling and optimize customer experience.

Your Mission

Implement a real-time inventory synchronization system that updates across all channels (online, mobile app, in-store POS) within 30 seconds of any transaction.

Technical Requirements

  • WebSocket connections for real-time updates
  • Event-driven architecture with message queues
  • Conflict resolution for simultaneous updates
  • Fallback mechanisms for network failures
  • Audit trail for all inventory changes

Claude Code Prompt Example

Create a real-time inventory sync system for AutoZone's e-commerce platform. 

Requirements:
- Handle 10,000+ concurrent users
- Sync inventory across 6,000+ stores
- Update within 30 seconds
- Handle network failures gracefully
- Include conflict resolution for simultaneous purchases

Tech stack: Node.js, Redis, PostgreSQL, WebSockets

Please implement the core inventory service with real-time sync capabilities.

Learning Objectives

  • Event-driven architecture patterns
  • Real-time data synchronization
  • Distributed system challenges
  • Performance optimization techniques

Scenario 2: Automated Reordering System

Intermediate 6-8 hours

Background

AutoZone needs to maintain optimal inventory levels while minimizing carrying costs. The system should predict demand and automatically trigger reorders.

Your Mission

Build an intelligent automated reordering system that uses historical sales data, seasonal trends, and external factors to optimize inventory levels.

Technical Requirements

  • Machine learning models for demand forecasting
  • Integration with supplier APIs
  • Business rules engine for approval workflows
  • Cost optimization algorithms
  • Exception handling for out-of-stock scenarios

Claude Code Prompt Example

Design an automated reordering system for AutoZone that:

1. Analyzes 3 years of sales history
2. Considers seasonal trends (winter tires, summer AC parts)
3. Integrates with supplier ordering APIs
4. Handles minimum/maximum order quantities
5. Includes manual override capabilities
6. Sends alerts for unusual demand patterns

Create the demand forecasting engine and reorder logic using Python/scikit-learn with a REST API interface.

Scenario 3: Multi-Location Inventory Allocation

Advanced 8-10 hours

Background

When customers order online, AutoZone needs to intelligently decide which store(s) should fulfill the order to minimize shipping costs and delivery time.

Your Mission

Create a sophisticated allocation engine that considers inventory levels, shipping costs, delivery speed, and store capacity constraints.

Technical Requirements

  • Optimization algorithms for allocation decisions
  • Real-time inventory availability checks
  • Geographic proximity calculations
  • Shipping cost APIs integration
  • Load balancing across distribution centers
  • Split shipment handling

Claude Code Prompt Example

Build an intelligent inventory allocation system for AutoZone e-commerce orders:

Scenario: Customer in Dallas orders 4 items:
- Oil filter (available at 12 nearby stores)
- Brake pads (available at 3 stores)
- Car battery (available at 8 stores)  
- Air freshener (available at all stores)

Optimize for:
1. Minimum shipping cost
2. Fastest delivery
3. Fewest shipments
4. Store capacity constraints

Implement allocation engine with constraint optimization, considering store hours, shipping zones, and inventory reservations.

Scenario 4: Complex Order Workflow Engine

Intermediate 5-7 hours

Background

AutoZone orders can be complex: some items ship from stores, others from warehouses, professional installation services, special orders, and commercial accounts with NET 30 terms.

Your Mission

Build a flexible order workflow engine that handles all order types and automatically routes orders through appropriate fulfillment processes.

Order Types to Handle

  • Standard retail orders (ship to home)
  • Buy online, pick up in store (BOPIS)
  • Professional installation orders
  • Commercial account bulk orders
  • Special order items (not in stock)
  • Warranty replacement orders

Claude Code Prompt Example

Create a flexible order workflow engine for AutoZone that handles these scenarios:

1. Regular customer orders brake pads + installation
   - Route pads to store inventory
   - Schedule installation appointment
   - Send confirmation with appointment details

2. Commercial customer bulk order (500 oil filters)
   - Check credit limit and terms
   - Route to warehouse fulfillment
   - Generate delivery schedule
   - Send invoice for NET 30 payment

3. Special order for rare vintage car part
   - Create supplier purchase order
   - Set customer expectations (2-3 weeks)
   - Track supplier delivery
   - Notify customer when available

Design using workflow engine pattern with configurable business rules.

Scenario 5: Black Friday Performance Optimization

Advanced 10-12 hours

Background

Black Friday generates 20x normal traffic. The order processing system must handle massive spikes while maintaining sub-second response times and preventing overselling.

Your Mission

Prepare the order processing system for Black Friday by implementing caching, queue management, and circuit breakers to handle extreme load.

Claude Code Prompt Example

Optimize AutoZone's order processing system for Black Friday:

Expected load:
- Normal: 1,000 orders/hour
- Black Friday: 20,000 orders/hour (8am-10am peak)
- 50,000 concurrent users
- Hot deals: 500 brake pad sets at 50% off

Implement:
1. Order queue management with priority lanes
2. Inventory reservation system (hold items for 10 minutes)
3. Circuit breakers for payment processing
4. Auto-scaling triggers
5. Real-time monitoring dashboards
6. Graceful degradation strategies

Focus on preventing cart abandonment and ensuring hot deal fairness.

Intermediate 6-8 hours

Background

AutoZone customers need to find parts that fit their specific vehicle. The compatibility engine must handle complex relationships between vehicles, parts, and fitment data.

Your Mission

Build a sophisticated vehicle compatibility system that accurately matches parts to vehicles and suggests alternatives when exact matches aren't available.

Claude Code Prompt Example

Create a vehicle compatibility engine for AutoZone:

Customer searches for: "brake pads for 2018 Honda Civic LX"

System must:
1. Parse vehicle details (year, make, model, trim)

Scenario 8: Fraud Detection System

Advanced 8-10 hours

Background

AutoZone processes millions of dollars in online transactions. A sophisticated fraud detection system is crucial to prevent chargebacks while minimizing false positives.

Your Mission

Build a real-time fraud detection system that analyzes transaction patterns and user behavior to identify suspicious activity.

Claude Code Prompt Example

Create a fraud detection system for AutoZone payments:

Analyze these risk factors:
1. Transaction velocity (multiple orders in short time)
2. Geographic anomalies (IP vs billing address)
3. High-value orders from new accounts
4. Unusual product combinations
5. Payment method patterns
6. Device fingerprinting

Implement real-time scoring with machine learning models. Include manual review queue for borderline cases and merchant rules for immediate blocking/approval.

Target: <0.1% false positive rate, >95% fraud detection rate
